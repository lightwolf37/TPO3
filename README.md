# Отчёт по тестированию ПО — Веб-приложение “Steam”

## 1. Введение
**Цель работы:**  
Провести системное тестирование платформы Steam для проверки корректности работы основного функционала, выявления дефектов и оценки соответствия требованиям пользователя. 

**Краткое описание программы:**  
Steam — это цифровая платформа для распространения игр и программного обеспечения, разработанная компанией Valve Corporation.

Приложение позволяет пользователю:

- приобретать и загружать игры
- управлять библиотекой игр
- запускать и обновлять игры
- общаться с друзьями через встроенный чат
- использовать торговую площадку (Торговая площадка)
- получать достижения и отслеживать игровую активность
- устанавливать модификации через Мастерскую

Ссылка на сайт:  
[https://store.steampowered.com/](https://store.steampowered.com/)

**Обоснование тестирования:**  
Системное тестирование необходимо для оценки работы Steam с точки зрения конечного пользователя, включая:

- проверку стабильности клиента и веб-интерфейса
- корректность работы ключевых функций (покупка, загрузка, запуск игр)
- соответствие функционала заявленным требованиям
- выявление багов интерфейса и производительности
- проверку взаимодействия между различными компонентами системы (клиент, сервер, сеть)

Учитывая сложность платформы и наличие регулярных обновлений, системное тестирование является критически важным для обеспечения стабильной и безопасной работы сервиса.

## 2. Тест-план (Test Plan)

### 2.1 Цели и задачи
- Проверка корректности установки и запуска игр
- Проверка работы библиотеки и управления играми
- Проверка функционирования магазина (покупка игр, добавление товара в корзину)
- Проверка работы друзей и чата
- Проверка корректного отображения интерфейса
- Выявление ошибок и дефектов
- Проверка соответствия заявленному функционалу

### 2.2 Область применения
**Тестируемые модули:**  
- Модуль установки и обновления игр
- Модуль библиотеки
- Модуль магазина 
- Модуль друзей и чата 
- Модуль профиля пользователя
- Модуль загрузок  

**Ограничения:**  
- Тестирование проводится только на десктопной версии в Steam  
- Не тестируются мобильные версии и альтернативные браузеры  

### 2.3 Стратегия тестирования
- Функциональное тестирование всех модулей  
- Регрессионное тестирование после исправления багов  
- Тестирование GUI для проверки корректного отображения интерфейса  
- Smoke-тестирование при каждом запуске приложения  
- Тестирование производительности (тест загрузки и отклика интерфейса)

### 2.4 Критерии начала тестирования
- Клиент Steam правильно установлен 
- Пользователь может авторизоваться или зарегистрироваться 
- Есть интернет соединение
- Доступен весь функционал тестируемых модулей (Библиотека, Магазин, Профиль)

### 2.5 Критерии завершения тестирования
- Выполнено ≥ 90% тест-кейсов  
- Все критические баги зарегистрированы и описаны  
- Программа соответствует требованиям пользователя
- Основной функционал работает

### 2.6 Тестовая среда
- **ОС:** Windows 10 Pro (64-bit)  
- **Клиент:** Steam v1773426488  
- **Интернет:** стабильное подключение  
- **Дополнительно:** 
	- имеющейся аккаунт
	- тестовая игра
	- инструмент для скриншотов и видеозаписи
### 2.7 Критические области
- Процесс установки и обновления игр
- Работа загрузок (Управление загрузок)
- Покупка игр и работа магазина
- Отображение библиотеки
- Работа друзей и чата
### 2.8 Критерии приостановки и возобновления тестирования
**Приостановка тестирования:**
- Клиент Steam не запускается
- Отсутствует интернет-соединение
- Критические ошибки (краши, невозможность авторизации)

**Возобновление тестирования:**
- Ошибки устранены
- Система работает стабильно
- Восстановлен доступ ко всем модулям
### 2.9 Роли и обязанности
- **Тестировщик:** выполнение тест-кейсов, фиксация багов
- **Разработчик:** исправление дефектов
- **Пользователь:** проверка удобства и интерфейса
### 2.10 Глоссарий
- **Steam** — платформа цифровой дистрибуции
- **Баг** — ошибка в работе программы
- **Тест-кейс** — сценарий проверки функции
- **SRS** — документ требований к системе
- **GUI** — графический интерфейс пользователя

### 6.1 Виды тестирования (по методичке)

В ходе тестирования платформы Steam были применены следующие виды тестирования:

#### Функциональное тестирование
Проверка работы основных функций (установка, запуск игр, покупка, чат).  
Пример: TC-01, TC-02, TC-07

#### Регрессионное тестирование
Проверка, что ранее работающий функционал не нарушен.  
Пример: повторный запуск игры после обновления

#### Smoke-тестирование
Быстрая проверка работоспособности системы после запуска.  
Пример: запуск Steam + проверка входа в аккаунт

#### GUI-тестирование
Проверка корректного отображения интерфейса.  
Пример: TC-13, TC-14

#### Тестирование производительности
Оценка скорости загрузок и отклика интерфейса.  
Пример: TC-11 — нестабильная скорость загрузки

#### Исследовательское тестирование

Свободное тестирование без сценариев для поиска багов.  
Пример: случайное переключение вкладок -> найден баг библиотеки

- В ходе тестирования выявлены ошибки, связанные с загрузками, редактированием профиля и поиском игр в библиотеке
